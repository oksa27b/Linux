Лекция
Какие у нас есть сетевые интерфейсы   
oksana@oksana-VirtualBox:~$ ip a                        
2 сетевых интерфейса это lo локальный интерфейс, который не выходит за пределы нашего компа
Link это макадрес
Inet  это наш айпи 10.0.0.1 с маской подсети /24
oksana@oksana-VirtualBox:~$ ip -s a  посмотрим статистику по пакетам, сколько было передано пакетов, какие-то низкоуровневые ошибки
oksana@oksana-VirtualBox:~$ ip r  далее информация по маршрутизации  (видим  3 маршрута самый главный маршрут первый он называется дефолт- это шлюз, через который мы отправляем пакеты за пределы нашей локальной сети и адрес шлюза указан в этой строке. И это адрес -точка, через который мы можем пойти дальше в другие сети.
Второй маршрут это и есть наша локальная сеть, адрес является точкой, через которую мы можем пойти дальше по нашим сетям, например в  интернет, наши соседние сети, все что не является нашей локальной сетью. Второй маршрут это как раз и есть наша локальная сеть, здесь мы можем ходить напрямую через свой интерфейс, доступный без мршрутизатора.
Третий маршрут это для внутренней сети, в нем можно увидеть метрики это условная стоимость маршрута, если у вас несколько маршрутов вы можете выбирать более дешевые.

root@oksana-VirtualBox:/home/oksana# iptables -I -m state –state ESTABLISHED,RELATED – j ACCEPT разрешить входящие пакеты от тех соединений, которые установил сервер с другими хостами в сети. Подключает к модулю стэйт, свойства ставят два обозначения, это пакеты которые будут возвращаться к нам с других серверов, к которым будем обращаться

root@oksana-VirtualBox:/home/oksana# iptables -L -nv посмотрим список правил 
что будет, если ваше соединение у вас разорвется, думайте об этом заранее
root@oksana-VirtualBox:/home/oksana# iptables -P INPUT DROP устанавливаем политику по умолчанию, если этого наша политика безопасности ссш работает, значит мы не нарушили свое собственное соединени.
root@oksana-VirtualBox:/home/oksana# iptables -L -nv посмотрим список правил, все правила разрешить статус акцепт, дроп означает сброс пакета без каких-либо действий. 
Проверка
root@oksana-VirtualBox:/home/oksana# iptables -L -nv  -t nat
все правила которые у нас есть временные, правила есть работают, до первой перезагрузки

root@oksana-VirtualBox:/home/oksana# iptables-save > ip.rules
root@oksana-VirtualBox:/home/oksana# cat ip.rules
root@oksana-VirtualBox:/home/oksana# iptables-restore < ip.rules считать это файл и восстановить из него все что у нас было, вся наша политика будет сброшена 
мы можем поставить несколько пакетов, чтобы при загрузке сохранялись автоматически
root@oksana-VirtualBox:/home/oksana# iptables-persistent netfilter-persistent
 при установке нам предлагает сохранить файл, мы да сохраняем

дальше устанавливаем сервис, чтобы при загрузки проходило все автоматически
